Mastering Git Stash & Local Cleanliness

Learning Objectives

  • Understand the concept and utility of stashing uncommitted work.
  • Master managing the Stash Stack using apply, pop, and drop.
  • Inspect, preview, and review stashed changes safely.
  • Clean untracked files and directories using git clean safely.
  • Apply best practices for context switching without creating incomplete commits.

1. Introduction

While working on a feature, you may need to switch branches or handle an urgent bug before your current work is complete. Instead of creating an incomplete commit, Git provides Git Stash, which temporarily saves your uncommitted changes. This lesson also covers Git Clean, a command used to remove unnecessary untracked files and keep your working directory organized.

2. What is Git Stash?

Git Stash takes your uncommitted changes (both staged and unstaged), saves them on a temporary stack, and reverts your working directory back to the last clean HEAD commit state.

Dirty Working Directory
↓ (git stash)
Stored on Stash Stack
↓ (git stash apply / pop)
Restored Working Directory

This allows you to switch branches safely or work on hotfixes without losing your unfinished progress.

3. Creating & Viewing Stashes

Creating a Stash

Save your active, uncommitted changes to the stash stack:

git stash

Viewing the Stash Stack

List all saved stashes stored in your local repository:

git stash list

Example Output:

stash@{0}: WIP on main
stash@{1}: WIP on feature-login

Each stash entry is indexed in a Last-In, First-Out (LIFO) stack format, with stash@{0} being the most recent.

4. Applying, Popping & Inspecting Stashes

Applying a Stash

Re-apply the most recent stash without removing it from the stash stack:

git stash apply

Target a specific stash in the stack:

git stash apply stash@{1}

Popping a Stash

Restore the latest stash and immediately remove it from the stack in a single step:

git stash pop

Inspecting Stash Contents

View a summary of modified files inside the top stash:

git stash show

Inspect exact line-by-line diffs within the stash:

git stash show -p

5. Removing Stashes

Delete a specific stash from the stack manually:

git stash drop stash@{0}

Wipe the entire stash stack completely:

git stash clear

6. Cleaning Untracked Files with Git Clean

Git Stash primarily manages tracked files. To remove untracked files and build artifacts cluttering your workspace, use git clean.

1. Preview Mode (Dry Run)

Always perform a dry run first to preview which files will be deleted:

git clean -n

2. Removing Untracked Files

Permanently remove untracked files from the working directory:

git clean -f

3. Removing Untracked Files & Directories

Recursively remove untracked files as well as untracked folders:

git clean -fd

Warning: git clean operation is destructive and permanently deletes untracked files. Always preview with -n first!

7. Key Commands Reference

Command Purpose
git stash Save uncommitted working changes to the stash stack
git stash list Display all stashed snapshots on the stack
git stash apply Restore stashed changes while keeping the stash on the stack
git stash pop Restore stashed changes and remove them from the stack
git stash show -p Inspect detailed line differences inside a stash
git stash drop <stash_id> Delete a specific stash entry
git stash clear Remove all saved stashes completely
git clean -n Preview untracked files that would be deleted (Dry run)
git clean -f Force delete untracked files permanently
git clean -fd Force delete untracked files and directories permanently

8. Industry Best Practices

  • Use git stash to handle context switches rather than creating broken or temporary commits.
  • Review stash contents with git stash show -p before applying them back to your workspace.
  • Always execute a dry run using git clean -n prior to running destructive deletions.
  • Clear or drop stashes regularly after popping them to keep your stash stack clutter-free.
  • Avoid storing long-term work in the stash; create a proper feature branch if changes will be paused for long periods.

Summary

In this lesson, you learned how to temporarily save unfinished work using Git Stash, restore it later with git stash apply or git stash pop, inspect and manage stashes, and clean untracked files using Git Clean. These commands help you switch tasks efficiently while keeping your working directory clean and organized.
