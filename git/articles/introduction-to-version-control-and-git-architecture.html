Introduction to Version Control & Git Architecture

Learning Objectives

  • Understand the core necessity of version control in software development.
  • Distinguish between Local, Centralized, and Distributed Version Control Systems.
  • Master Git's Three-Tree Architecture model.
  • Configure global user options and initialize a project repository.
  • Identify the primary file states throughout the Git lifecycle.

1. Introduction

Software development involves continuous changes to files and code. Without a proper system to track these changes, managing projects becomes difficult, especially when working in teams. Git is a Distributed Version Control System (DVCS) that helps developers track changes, collaborate efficiently, and restore previous versions whenever needed.

2. What is Version Control?

A Version Control System (VCS) records changes made to files over time, allowing developers to:

  • Track project history chronologically
  • Restore previous versions safely
  • Collaborate seamlessly with multiple developers
  • Prevent accidental data loss

3. Types of Version Control Systems

1. Local Version Control

Stores project versions as simple patches on a single computer's local disk space.

2. Centralized Version Control (CVCS)

Uses a central server where all project versions are stored. Developers must connect directly to this single server to check out files and collaborate.

3. Distributed Version Control (DVCS)

Every developer clones a complete copy of the repository, including its full historical timeline, onto their local machine. Git is a Distributed Version Control System.

4. Why Choose Git?

Git is widely used across modern software engineering environments because it is:

  • Fast and computationally efficient
  • Capable of working offline without constant server access
  • Built around lightweight branching and merging workflows
  • Enables easy collaboration and distributed contribution
  • Maintains complete project history cryptographically

4. Git's Three-Tree Architecture

Git manages files through three primary structural areas within your local machine:

Working Directory
↓ (git add)
Staging Area (Index)
↓ (git commit)
Local Repository

Tree Definitions

  • Working Directory: The local file directory where you create, modify, and edit files.
  • Staging Area (Index): A temporary hold area that previews selected changes before committing.
  • Local Repository: The permanent location storing committed snapshots (commits) inside the hidden .git folder.

5. Installation & Configuration

Install Git for your operating system and verify the installation via the command line:

git --version

If a version number is displayed, Git is installed successfully.

Configuring User Identity

Set your global user identity before creating commits:

git config --global user.name "Your Name"
git config --global user.email "your@email.com"

To inspect your current configuration settings:

git config --list

6. Creating Your First Repository

To start tracking a project with Git, navigate to your target folder and initialize a new repository:

mkdir MyProject
cd MyProject
git init

This creates a hidden .git folder in the directory, initializing the local repository database.

7. File Lifecycle & States

Files in a repository transition across four primary states as you work:

Untracked
↓ (git add)
Staged
↓ (git commit)
Committed
↓ (Edit file)
Modified

State Overview

  • Untracked: The file is new in the working directory and Git is not tracking its changes.
  • Modified: A tracked file has changes that have not yet been staged.
  • Staged: Modified or new files added to the staging area, ready to be committed.
  • Committed: File state is securely saved into the local repository snapshot.

Checking Repository Status

Inspect the state of tracked and untracked files at any time using:

git status

8. Key Commands Reference

Command Purpose
git --version Check Git installation version
git config --global user.name Set default username globally
git config --global user.email Set default email globally
git config --list View active Git configurations
git init Initialize a new Git repository
git status Check current status of the working tree
